将hexo部署迁移至zeit

2021年02月13日 304 字 大概 1 分钟
  1. 1. 关于zeit.co
  2. 2. 步骤
    1. 2.1. 域名解析

Github Page在国内访问较慢，可以将服务部署到静态站zeit.io, 每月20G免费流量，并且能同步github项目。

关于zeit.co

这天在搜索资料时，注意到有博主提到 zeit.co，一个静态网页和无状态函数托管的云平台。试用了之后，感觉非常强大，它能够以 Github App 的形式与 GitHub 集成，接管 repo 的 CI，进行深度定制，并且能够快速部署、自动伸缩。

关键是，现在基本功能是免费的，这对于我们托管博客来说，已经很够用了。更惊喜的是，它还提供智能 CDN 和智能 DNS。

步骤

注册。打开登陆页面使用 GitHub 账号登陆即可。

导入。在主页面点击 “Import Project”，然后 “Import Project From Git Repo”，选择 GitHub。进行一下授权.

部署。导入过程中，选择 other 模板即可，默认 ci 命令不用改，Root Directory 也不用改，导入完成后就会自动帮你deploy。

最后在部署卡片中就可以看到 zeit 生成的 URL

域名解析

进入项目后选择settings选项，再选择Domains，进去后添加你拥有的域名即可
